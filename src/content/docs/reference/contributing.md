# Contributing

Contributions are more than welcome! There are many ways to support this project, whether through code, feedback, or financial support.

## Code Contributions

If you'd like to contribute code, please fork the repository, create a branch, and submit a pull request.

When submitting pull requests, please try to follow the existing code layout for internal consistency. I'm not particularly strict about formatting as long as it compiles, though I may ask you to tidy things up if it's especially messy.

## Reporting Issues

- **Feature requests:** If you have an idea for a feature but can't implement it yourself, please submit an issue and I'll add it to the backlog.
- **Bug reports:** If you find a bug, please report it as an issue with a description of how to replicate the problem.

[Click here to create a new FR or report an issue](https://github.com/swiftDialog/swiftDialog/issues/new/choose)

## Financial Contributions

If you find this project useful—particularly if it supports your work or business—please consider contributing financially. Your support helps ongoing development and maintenance.

- [GitHub Sponsors](https://github.com/sponsors/bartreardon)
- [Buy Me a Coffee](https://www.buymeacoffee.com/bartreardon)

Organisations using this project commercially are welcome to sponsor if they find value in it. There's no obligation, and it won't affect how I engage with anyone—but it does help keep the project moving forward.

## A Note on Kindness

This is a passion project maintained in my spare time. If you have suggestions for improvements or know a better approach to something, I'd genuinely love to hear about it—whether as a pull request, an issue, or just a comment. All I ask is that feedback be constructive and respectful.
