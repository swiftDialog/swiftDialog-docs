## What's New in swiftDialog 3.1.0

### Cards Workflow (Multi-Step Dialogs)

swiftDialog 3.1.0 introduces **Cards** — a powerful new feature for building multi-step dialog workflows from a single JSON configuration.

- Define an array of `cards` in your JSON, each with its own title, message, icon, and input fields
- Users can navigate **forward and backward** between cards, with input preserved across navigation
- Accumulated input from all cards is returned as a single JSON output on completion
- Global configuration properties (defined outside the `cards` array) act as defaults, overridable per card
- New `--onadvance` argument allows a shell script to run when the user advances between cards
- Card output can be piped to an **external processing script** for real-time handling
- Required field validation is enforced per card before advancing
- Button bar adapts automatically (Back / Next / Done) based on card position

### Separate Notification Apps

Notifications are now delivered by dedicated helper apps (`DialogNotifier-Alert.app` and `DialogNotifier-Banner.app`) rather than Dialog.app itself.

- Separate apps for **Alert** and **Banner** style notifications, ensuring correct notification presentation style
- Helper apps are properly code-signed and match the notification style required by macOS
- "Approve Notifications" alert now persists until dismissed (no longer auto-dismissed)
- New app icons for the notifier helper apps

> **Deprecation Notice:** Sending notifications via `Dialog.app` directly (without `--style alert` or `--style banner`) is deprecated. A warning is now printed to `stderr` when this legacy usage is detected. This behaviour will be removed in a future release. Use the dedicated notifier apps for new deployments.
